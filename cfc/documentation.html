CFC Controller

The CFC Controller helps invoke ColdFusion Component (CFC) methods by building a path and POSTing form-like data. It supports declarative wiring via attributes such as data-cfc-path, data-cfc-method, and data-cfc-args. The controller normalizes default names and builds paths when needed.

Attributes

HTML Attributes

These are standard HTML attributes (with no prefix) used by Table Sort.

Attribute Element(s) Default Value(s) Description
aria-busy table
 false true
false
 The controller will set this to true any time it is sorting the table.
You can use the value in your CSS to indicate that work is in progress.
Note that if data-cfc-usebusy="false" is set, the controller will not toggle this attribute.

data- Attributes

All of these attributes are prefixed with "data-" in the attribute name.

Attribute Element(s) Default Value(s) Description
controller table
 cfc This attribute (on either element) provides the controller functionality.

data-cfc- Attributes

All of the attributes below are prefixed with data-cfc-.

Attribute Element(s) Default Value(s) Description
path (root) derived from data-cfc-name or current page (string)

required

The URL path to the CFC. If omitted, the controller will generate one from data-cfc-name or the current page name and ensure it ends with .cfc.
name any current page name (string) The logical CFC name; used to derive data-cfc-path when a full path is not provided.
method (any) (string)

The CFC method name to call. This could be placed on the root element or on the triggering element (for example, a button).

The controller could also have multiple child elements each with a different value for this so that buttons in that element use the specific method.
args (element with method attribute) query-string A query-string of key=value pairs (for example foo=bar&count=1) which will be parsed and merged with any runtime-collected args.
form form element or container with inputs (presence) Indicates that form elements inside the specified element should be collected as arguments to send to the CFC method.
arg (child elements inside method element) (string) Use data-cfc-arg="name" on child elements to include additional named arguments; their values are read with the controller's value resolution logic.
channel (root) (new uuid) (string)

The unique channel for this CFC instance; used to identify events.

Each CFC instance will have its own channel, allowing for external code to respond to events specific to that instance.
target / receiver (child elements inside method element) (string) data-cfc-target on the triggering element indicates the logical receiver name; elements with data-cfc-receiver="name" will have their textContent replaced with the response.
usebusy root

Controls whether the controller toggles the busy state on the method element during a call.

By default the controller will set aria-busy on the element that has data-cfc-method.
true
false Disables busy state toggling.
value (elements used as args) (string)

Specifies the value to use for the argument.

Order of precedence:

  • data-cfc-value
  • data-value
  • value
  • textContent
columns (root) (JSON object) JSON mapping used to rename/normalize columns when the controller parses CFC query results. Provide a JSON object mapping original column names to target names (a mapping value may be a string or an object with a name property). Example: {"ID":"id","FullName":"name"}.
column-case (root) lower (lower|upper|camel|preserve) Controls the case transformation applied to column names when no explicit mapping is provided. Default is lower. Other options: upper, camel, and preserve.
column-types (root) (JSON object) JSON map of column names to desired types. Supported types: int/integer, float/number, boolean, date, and string. Example: {"id":"int","active":"boolean"}. The controller will coerce parsed values when possible.

Events

These are custom events that the controller will fire when it takes certain actions.

cfc: Events

All of these events are prefixed by "cfc:".

Event Parameters Description
calling

detail.channel: (string) — the unique channel for this CFC instance

detail.path: (string) — the URL path to the CFC

detail.method: (string) — the CFC method being invoked

detail.args: (object|FormData) — the arguments that will be sent

 Fired immediately before the controller performs the HTTP request. Useful for showing loading states or inspecting the outbound request.
called

detail.channel: (string) — the unique channel for this CFC instance

detail.path: (string)

detail.method: (string)

detail.args: (object|FormData)

detail.response: (Response) — the fetch Response object

detail.result: (any) — the parsed or normalized result (JSON/object or text)

 Fired after a response is received and processed (parsed/normalized). The detail.result contains the value returned by the controller.

Methods

These methods are available on the controller and can be called programmatically.

Method Arguments Description
call (Event) OR (path, method[, args]) Primary entry point. If called with an event, it will extract attributes from the event target. With a path and method it will call the specified CFC method directly.
callEvent Event Handles an event-triggered call: prevents default, sets busy flag, gathers args, calls CFC method, and passes the response to handleResponse.
callPathMethod path, method, args Helper that delegates to runCfcMethod and returns the result.
runCfcMethod path, method, args Performs a fetch to path?method=<method> using POST and returns parsed JSON or text. Throws on HTTP errors or ColdFusion error responses.
getArgs element Collects args from data-cfc-args, child data-cfc-arg elements, and optionally from form fields within the method element.
collectFormData element Collects input values from a form or container, handling checkboxes and radio groups appropriately.
getElementValue element Resolves a value using the hierarchy: data-cfc-value, data-value, element.value, or textContent.
handleResponse response, event Finds receiver elements matching data-cfc-receiver and places the response text into their textContent.
parseQueryString qs Parses a query-string into an object, supporting repeated keys as arrays.
