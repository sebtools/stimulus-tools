Styler Controller

The Styler Controller keeps a target element's inline styles in sync with a source control (select or radio inputs). It reads a data-styles attribute from the selected option or checked radio and applies those rules to the target element's style attribute while removing previously applied rules managed by the controller.

Keep in mind that while any styles can be impacted with styler, the recommended approach would be to use it only to change custom property values and have other code drive off of those custom property values.

Targets

Indicated by data-styler-target or the Stimulus targets API.

Target Element(s) Description
source select or a container with radio inputs The control used to choose styles. For a <select> the selected option's data-styles is read. For radio groups, the checked radio's data-styles is used.
target any element The element whose inline style attribute will be modified.

Attributes

data- Attributes

These are standard data- attributes used by the controller (no controller-specific prefix).

Attribute Element(s) Default Value(s) Description
controller any element styler Add data-controller="styler" to enable the controller on an element.
styles option
input[type="radio"]
target element (see data-styler-target) 		(CSS rules string) Holds CSS rules such as color: red; background: yellow;. Placed on source controls (option or radio) to define the styles to apply, and written to the target element's data-styles to trigger the update.

data-styler- Attributes

Attributes using the controller prefix are typically used for Stimulus targets and controller-specific configuration.

Attribute Element(s) Default Value(s) Description
target any element source
target		 Declare Stimulus targets using data-styler-target="source" and data-styler-target="target". The source is the control that supplies the styles (a <select> or radio group). The target is the element whose inline style will be updated.

Methods

Method Description
updateFromSource() Reads the selected option or checked radio's data-styles and writes it to the target's data-styles.
applyTargetStyles(oldStyles, newStyles) Removes the CSS properties found in oldStyles from the inline style and appends the rules from newStyles.

Notes

  • The controller updates inline styles; if your app uses CSS classes or external stylesheets you may prefer a different approach.
  • The controller relies on a simple parsing strategy: it splits style strings on ; and treats the left side of each rule as the property to remove when updating.
  • If multiple controllers or code paths update the same inline style, mixing approaches could lead to unexpected results; prefer keeping the styler's rules isolated or prefixed in your styling strategy.

Using CSS custom properties

In many cases it's convenient for the styler to set CSS custom properties (variables) on the target and let your stylesheet consume them. This keeps the actual style rules in CSS while the controller only sets data values such as --accent: #ff0000;.

Example: set data-styles="--accent: #ff0000; --bg: #fff0f0;" and in your stylesheet use color: var(--accent) or background: var(--bg). The controller will write the variables into the element's inline style, which makes them available to descendant rule lookups.
