RecordCFC Controller

The RecordCFC routes data from the record component to a CFC via the cfc component. It wires a record UI controller to server-side CFC methods via the cfc controller. It listens for record UI events (add/update/delete), translates those into CFC calls, and then fires data events back into the page when the server responds.

Prerequisites

  • This controller expects both a record controller and a cfc controller to be attached to the same element. It will throw an error on connect if either is missing.

Values

The controller defines the following Stimulus value:

Value Attribute Description
idarg data-recordcfc-idarg-value The CFC parameter name used to pass the record ID to server methods.

Attributes

The RecordCFC controller relies on a set of attributes (typically placed on the same element as the record controller) to configure which CFC methods to call for record actions.

Attribute Element(s) Description
data-recordcfc-method-save container / method element Name of the CFC method to call when a record is saved/updated or used for add/save operations (controller reads per-action names).
data-recordcfc-method-add container / method element Name of the CFC method to call when creating a new record (optional if save covers both).
data-recordcfc-method-delete container / method element Name of the CFC method to call when deleting a record.
data-recordcfc-idarg container / method element Name of the parameter to use when sending the record ID to the CFC (e.g. id or recordId). The controller will throw if this is not configured where required.

Events handled

The controller listens for these document-level UI events emitted by the record controller:

Event Parameters Description
record:ui:update detail: { table, id, record, element } Triggers the configured CFC save method to update an existing record.
record:ui:add detail: { table, record, element } Triggers the configured CFC save/add method to create a new record. Expects the server to return an id for the new record.
record:ui:delete detail: { table, id, element } Triggers the configured CFC delete method to remove a record on the server.

Events emitted

After performing CFC calls the controller will emit document-level data events that the record controller consumes:

Event Parameters Description
record:data:update detail: { table, id } Fired after a successful update; includes the id (returned id preferred).
record:data:add detail: { table, id, record } Fired after a successful add. Note: server MUST return an id for new records.
record:data:delete detail: { table, id } Fired after a successful delete.
record:data:error:<action> detail: { error, originalEvent, element } Fired when a CFC call fails for an action (add/update/delete) with error details.

Behavior and sequence

  1. On connect the controller finds the co-located record and cfc controllers.
  2. It subscribes to document-level record:ui events and ignores events outside the element's scope.
  3. When an event occurs it constructs parameters via buildCfcParams, sets a loading state (aria-busy), calls the CFC method using the cfc controller and then emits the appropriate record:data: event.

Methods

Method Description
getIdArgName(element) Finds the configured ID argument name via data-recordcfc-idarg (throws if missing where required).
getMethodName(element, action) Reads the configured method name for an action (e.g. save, delete) from data-recordcfc-method-<action>.
buildCfcParams(record, idArgName, id) Returns the parameters object to send to the CFC; merges record fields and attaches the id under the configured id argument name when provided.
fireDataEvent(action, detail, element) Dispatches record:data:<action> with the provided detail and element to inform the record controller of server-side changes.
fireErrorEvent(action, error, originalDetail) Dispatches record:data:error:<action> with error information so callers can react to failures.
setLoadingState(isLoading) Sets aria-busy on the element and optionally disables/enables form controls while the CFC call is in progress.

Usage example

<div data-controller="record recordcfc cfc" data-recordcfc-idarg-value="id" data-recordcfc-method-save="saveRecord" data-recordcfc-method-delete="deleteRecord">
<!-- inner record markup here -->
</div>

Notes

  • The controller delegates the actual HTTP/CFC integration to the cfc controller via its call method.
  • Ensure your CFC save/add method returns an id when creating new records; the controller requires this to fire the add data event.
  • The controller intentionally scopes events to elements contained within its root element to avoid handling unrelated record events on the page.
