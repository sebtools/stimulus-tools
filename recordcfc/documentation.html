RecordCFC Controller

The RecordCFC routes data from the record component to a CFC via the cfc component. It wires a record UI controller to server-side CFC methods via the cfc controller. It listens for record UI events (add/update/delete), translates those into CFC calls, and then fires data events back into the page when the server responds.

Prerequisites

  • This controller expects both a record controller and a cfc controller to be attached to the same element. It will throw an error on connect if either is missing.

Attributes

Attribute Element(s) Default Value(s) Description
aria-busy container / table false true, false Indicates that work is in progress for the controller on this element. Other controllers may observe this attribute to detect busy state.

data- (controller attribute)

These are standard HTML attributes (with no prefix) used by RecordCFC.

data-recordcfc- Attributes

All of these attributes are prefixed with "data-recordcfc-" in the attribute name.

Attribute Element(s) Default Value(s) Description
method-save container / method element (string) Name of the CFC method to call when saving/updating or as the generic save/add method.
method-add container / method element (falls back to method-save) (string) Name of the CFC method to call when creating a new record. Optional — if omitted the method-save method will be used for adds.
method-delete container / method element (string) Name of the CFC method to call when deleting a record.
method-get table element (string) Name of the CFC method used to fetch a single record (when the table element has a data-record-id).
method-gets table element (string) Name of the CFC method used to fetch multiple records (bulk get/load).
method-load table element (falls back to method-gets) (string) Alternative name for the bulk load method; used if method-gets is not present.
idarg container / table / method element (string) Name of the parameter to use when sending the record ID to the CFC (for example WidgetID, id, or recordId).
args container / table element (JSON string | query string) Static arguments to include on load calls. Can be JSON or a URL query string (falls back to data-record-filter if present).

Note: attributes may be placed at the container element or at nested table/method elements; the controller uses the record controller's attribute-resolution helpers to locate attributes in the correct record/table scope.

Events handled

The controller listens for these document-level UI events emitted by the record controller:

Event Parameters Description
record:ui:update detail: { table, id, record, element } Triggers the configured CFC save method to update an existing record.
record:ui:add detail: { table, record, element } Triggers the configured CFC save/add method to create a new record. Expects the server to return an id for the new record.
record:ui:delete detail: { table, id, element } Triggers the configured CFC delete method to remove a record on the server.

Events emitted

After performing CFC calls the controller will emit document-level data events that the record controller consumes:

Event Parameters Description
record:data:load detail: { table, records, origin?, element? } Fired after a successful load/get; records is an array normalized to objects shaped { id, record } where id is either the configured id-argument value on the server-returned object or record.id. The record controller expects this normalized form.
record:data:update detail: { table, id } Fired after a successful update; includes the id (returned id preferred).
record:data:add detail: { table, id, record } Fired after a successful add. Note: server MUST return an id for new records.
record:data:delete detail: { table, id } Fired after a successful delete.
record:data:error:<action> detail: { error, originalEvent, element } Fired when a CFC call fails for an action (add/update/delete) with error details.

Methods

Method Description
getIdArgName(element) Resolves the configured ID argument name by locating the nearest data-recordcfc-idarg attribute in scope (throws if missing where required). Demos typically set this attribute on the container or table element.
getMethodName(element, action) Reads the configured method name for an action (e.g. save, delete) from data-recordcfc-method-<action>.
