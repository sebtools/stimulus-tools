EventLog Controller

The EventLog Controller is a small debugging utility that listens for custom events and renders a readable log of the last events (with pretty-printed JSON details). It's handy for development and inspecting events emitted by other controllers.

Attributes

data- Attributes

These are standard data- attributes (no controller-specific prefix).

Attribute Element(s) Default Value(s) Description
controller any element eventlog Add data-controller="eventlog" to enable the EventLog controller on an element.

data-eventlog- Attributes

All of these attributes are prefixed with "data-eventlog-" in the attribute name.

Attribute Element(s) Default Value(s) Description
events element with controller space-separated event names A space-separated list of event names that the controller should listen for on document. For example: data-eventlog-events="record:ui:added record:ui:update".

Methods

Method Arguments Description
addEventListener eventName (string) Adds an event name to the data-eventlog-events list (if not present) and starts listening for it on document.
clear none Clear the log contents and reset the internal event counter.

Notes

The controller listens on document so it will see events dispatched from anywhere in the page. The event names are matched exactly — include the full namespaced event string (for example record:ui:added).
