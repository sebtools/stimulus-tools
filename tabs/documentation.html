Tabs Controller

The Tabs controller provides accessible tabbed UI behavior using ARIA roles. It wires up click and keyboard navigation (left/right arrows) for elements with role="tablist", role="tab", and role="tabpanel".

Attributes

HTML / ARIA attributes

Attribute Element(s) Description
role="tablist" container Wraps the tab buttons.
role="tab" button, element Represents a selectable tab. The controller expects tab elements to live directly under the tablist (top-level).
aria-selected role="tab" Set to true on the active tab. The controller will update this attribute when tabs are selected.
aria-controls role="tab" References the id of the panel (role="tabpanel") that the tab controls.
role="tabpanel" panel Content panel associated with a tab. Panels that are not active should include the hidden attribute.

data- Attributes

Attribute Element(s) Default Value(s) Description
controller * tabs Attach the tabs behavior.

data-tabs Attributes

Attribute Element(s) Default Value(s) Description
all role="tab" false true | (absent) If the selected tab has data-tabs-all="true" the controller will show all panels instead of only the one referenced by aria-controls.
name / data-name role="tab" string Optional name used when calling methods such as selectTab(name). If not present the tab's textContent is used.

Methods

You can call these methods on the element's controller instance (the controller attaches itself to the element):

Method Arguments Description
selectTab(tab) element or name Selects the given tab (either the DOM element, or the tab's name as set via data-tabs-name/data-name or the tab text).
showSelectedPanel() Updates panels visibility based on the currently selected tab.
getTabButton(name) name Returns the tab element matching the provided name.

Accessibility notes

  • Ensure aria-controls on each tab references a valid panel id.
  • Set aria-selected="true" on the initially-active tab.
  • Panels that are not active should include the hidden attribute so assistive tech will ignore them.
